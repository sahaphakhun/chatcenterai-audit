# คู่มือย่อ: จุดลดช่องว่างในหน้า Dashboard V2 (/admin/dashboard)

ไฟล์อ้างอิงหลัก: `views/admin-dashboard-v2.ejs` (มี inline CSS) และ utility class ของ Bootstrap

## รายการพื้นที่ที่ลดได้พร้อมผลกระทบ

### 1) Header หน้า + แถบปุ่มนำเข้า/ส่งออก/สร้าง
- ค่าเดิม: `dashboard-header` padding บน-ล่าง 20px, margin-bottom 24px; ปุ่มใช้ `me-2` คั่นระยะด้านขวา
- ลดได้: ลด padding เหลือ 12–16px และ margin-bottom เหลือ 12–16px; ปุ่มเปลี่ยนเป็น `me-1` หรือใช้ `gap-2 flex-wrap`
- ผลกระทบ: เห็นการ์ดรายการเร็วขึ้นและลด “พื้นที่โล่ง” ตอนเปิดหน้า; ถ้าลดมากเกินไปปุ่มจะชิดกันจนกดพลาดง่าย โดยเฉพาะบนจอเล็ก

### 2) แถวค้นหา (search bar)
- ค่าเดิม: แถว `row mb-3` มีช่องว่างแนวตั้ง ~16px ก่อนเริ่ม editor card
- ลดได้: เปลี่ยนเป็น `mb-2` หรือเอา margin-bottom ออกเมื่ออยู่บนเดสก์ท็อป; ลด `g-3` ของแถวเป็น `g-2` หากเพิ่มปุ่ม/ตัวกรองในอนาคต
- ผลกระทบ: พื้นที่เหนือ editor card กระชับขึ้น เห็น editor หรือการ์ดชุดแรกเร็วขึ้น; ถ้าลดมากจะทำให้ header กับ search ติดกันจนแยกสายตายาก

### 3) Instruction Editor Card (เลือก/แก้ไข instruction)
- ค่าเดิม: `card ... mb-4` (margin-bottom ~24px), `card-body` padding Bootstrap (~16px), แถวภายใน `g-3`; ส่วนแจ้งเตือนใช้ `mt-3`
- ลดได้: ใช้ `mb-3`, ลด gutter เป็น `g-2`, ลด `mt-3` เป็น `mt-2` และลด padding ของ `card-body` ลง ~20% หากเนื้อหายังไม่ล้น
- ผลกระทบ: ลดการสกรอลเพื่อไปถึงรายการการ์ดมากขึ้น; หากลด padding มาก ฟอร์มจะดูแน่นและไอคอนเตือนอาจซ้อนกับข้อความบนมือถือ

### 4) การ์ด Instruction + รายการ Data Item
- ค่าเดิม: การ์ดแต่ละใบ `margin-bottom: 16px`; `instruction-header` padding 16px + `gap: 12px`; `data-items-container` padding 12px 16px; `data-item` padding 10px 12px + margin-bottom 8px; footer padding 12px 16px + `gap: 8px`; ปุ่ม list padding 4–8px
- ลดได้: ลด margin-bottom การ์ดเหลือ 10–12px, header/footer padding เหลือ 10–12px, container เหลือ 8–12px, data-item เหลือ 8–10px และ gap เหลือ 6–8px; บนมือถืออาจลด padding ปุ่มเหลือ 6px แนวตั้งแต่คงความสูง 40–44px เพื่อให้กดง่าย
- ผลกระทบ: เห็นจำนวนการ์ดและ data item ต่อหน้าจอมากขึ้น (สำคัญกับมุมมองมือถือ); ลดมากเกินไปทำให้ตัวเลขลำดับ/ป้าย type แน่น อ่านยาก และโอกาสกดปุ่มลบ/ย้ายผิดสูงขึ้น

### 5) Empty State
- ค่าเดิม: `.empty-state` padding 48px 24px, ไอคอน margin-bottom 16px
- ลดได้: ลด padding เหลือ 24–32px และ margin-bottom ไอคอนเหลือ 8px หรือเอาไอคอนออกในมุมมองมือถือ
- ผลกระทบ: เพิ่มพื้นที่ให้เนื้อหาอื่นเมื่อยังไม่มีข้อมูล; ถ้าตัดจนเหลือน้อยเกินไปจะดูเหมือนหน้าหายหรือระบบผิดปกติ (ขาดจุดพักสายตา)

### 6) Modal (Create/Edit/Preview/Upload Excel)
- ค่าเดิม: ใช้ค่า padding มาตรฐาน Bootstrap (~16px) ไม่มี override
- ลดได้: ไม่จำเป็นเร่งด่วน แต่ถ้าต้องการความกระชับบนจอเล็ก ลด padding ของ `.modal-body` เป็น ~12px และลด margin-bottom ของฟอร์ม (`mb-3` → `mb-2`)
- ผลกระทบ: ทำให้ข้อมูลพรีวิวเห็นได้มากขึ้นใน modal; หากลดเกินไปพื้นที่ระหว่างคอนโทรลจะน้อยและเพิ่มโอกาสแตะโดนผิดบนจอมือถือ

## แนวทางเลือกใช้
- ลดทีละขั้น (เช่น 20% ก่อน) แล้วทดสอบจอมือถือ + เดสก์ท็อป 13–14" เพื่อหาจุดสมดุลระหว่างพื้นที่ข้อมูลและระยะกดปุ่ม
- ให้ความสำคัญกับส่วนที่ใช้ซ้ำหลายใบ (การ์ด instruction และ data item) เพราะส่งผลกับ viewport สูงสุด
- รักษา hit area ของปุ่มไว้อย่างน้อย ~40px สูงในทุกการปรับ เพื่อลด miss-touch
